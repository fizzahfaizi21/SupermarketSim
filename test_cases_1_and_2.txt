# Test Cases for Stock Shelves and Customer Transaction

## Stock Shelves Test Cases

### Test Case ID: TC_SS_001
**Description:** Happy path - Successfully stock shelves with correct category and available inventory.
**Inputs:** Product: "Apple", Category: "Fruits", Quantity: 10, Current Inventory: 50
**Expected Results:** Inventory decreases by 10, shelves stocked successfully, no error message.
**Dependencies:** Inventory system, product database.
**Initialization:** Ensure product exists in inventory with sufficient quantity.
**Steps:**
1. Select "Stock Shelves" option.
2. Enter product name "Apple".
3. Enter category "Fruits".
4. Enter quantity 10.
5. Confirm stocking.

### Test Case ID: TC_SS_002
**Description:** Full shelf - Attempt to stock shelves when shelf is already at maximum capacity.
**Inputs:** Product: "Milk", Category: "Dairy", Quantity: 5, Shelf Capacity: 20, Current Shelf Stock: 20
**Expected Results:** Error message "Shelf is full", inventory unchanged, stocking fails.
**Dependencies:** Inventory system, shelf capacity tracking.
**Initialization:** Fill shelf to maximum capacity.
**Steps:**
1. Select "Stock Shelves" option.
2. Enter product name "Milk".
3. Enter category "Dairy".
4. Enter quantity 5.
5. Confirm stocking.

## Customer Transaction Test Cases

### Test Case ID: TC_CT_001
**Description:** Cash payment with correct change - Customer pays with cash and receives correct change.
**Inputs:** Total Amount: $15.50, Cash Tendered: $20.00
**Expected Results:** Change: $4.50, transaction successful, receipt generated.
**Dependencies:** Checkout system, cash handling.
**Initialization:** Start transaction with items totaling $15.50.
**Steps:**
1. Scan items totaling $15.50.
2. Select cash payment.
3. Enter cash amount $20.00.
4. Process payment.
